Inheritance

    Inheritance is one of the main techniques of object-oriented programming. It takes a regular class and extends it so that a more sophisticated class can be created. Suppose you have a bunch of different classes: an Employee class, a HourlyEmployee class, a SalariedEmployee class, a FullTimeHourlyEmployee class, and a ParttimeEmployee class. Each class represents a different kind of employee. Normally, you would of had to code each class separately. However with inheritance you can code one base class (Employee) and derive all of the other classes from that one base class.

Note: If you have a animal class and a person class, it does not make sense to use inheirtance. In the above case (of the employees) it does makes sense to use inheirtance because all of the classes are different kinds of employees.

    Inheirtance in programming is similiar to that of a child inheriting something from his or her parents. Usually, the child inherits all of the possessions of his or her parents so to the extended class (the child) inherits all of the attributes and methods from the base class (the parent’s). The syntax for creating a child class is: public class ClassName extends BaseClassName(){...}

    Inheritance is great, but there are a few things you must keep in mind:

  1. When you create an object from a derived class, the parent (the base class) also gets created. You cannot have a child without a parent.
  2. The parent always gets created before the child. Be careful about implementing constructors. (More on this later)
  3. If the parents’s attributes are private, you can not access / manipulate them directly even though you inherited them and they are your own. You must go to the parent and ask.
  4. A child can call any one of his parent’s methods.
  5. A child can use any one of his parent’s constructors. The syntax is: super(); (More on this later)
  6. A child can not call his or her parent’s attributes if they are private by using super.attribute() (even if the child inherited them and they are now his). The child has to go through the public methods (ask his or her parent for permission). (Look at rule 4)

The is-a relationship between a derived class and a base class.

  A derived class (a child class) is just a sophisticated version of the base class (the parent). This means if you pass a HourlyEmployee object inside a function taking an Employeee parameter, there will be no compiler error. A HourlyEmployee is just an Employee. However, if you pass an Employee object inside a function taking a HourlyEmployee parameter, you will get an error because an Employeee is not a HourlyEmployee. An employee could be an a hourly employee, a salaried employee, a full time hourly employee, a part time employee, etc.

The final Modifier

  If you add the word final to a method ( final public void MethodName() ), no one can come and override that method. Not even the extended class (ie the children).

  If you add the word final to a class ( final public class ClassName{} ), you can no longer extend that class.

To Recap: Taking a class and extending it so that other classes can be created is known as inheirtance and is one of the main techniques of object-oriented programming. The syntax is as follows: public class ClassName extends BaseClassName(){...}
