Overloading Methods Vs Overriding Methods

  When you use inheritance, the extended class inherits all of the attributes and methods from the base class. Some of these methods may be useless to the extended class. The extended class can fix this problem by either overloading the useless method or overriding them.

  Overriding does not mean destroying the parents's methods. It just means you are suppressing the parents’s methods. Overloading a method is not the same thing as overriding a method. An overridden method must look identical to the method it is overriding (same return type, same name, and same signature). An overloaded method has the same name but the parameters (signature) are different.

Overloaded Method

public Boolean equals (Object obj) // Original Method
public Boolean equals (ClassName a)// Overloaded Method (Different Parameters)

Note: When a derived class overloads a method, it still inherits the original method from the base class.

Overridden Method

public void setEmployeeName(String name) // Original Method (Base Class)
public void setEmployeeName(String name) // Overridden Method (Child Class)

Note: If you want to call a suppressed method (an overloaded method) you need to use the keyword super (super.methodName()). If you want to use the overridden method, you can call it directly because it is public and anyone can call it. For example if you have two methods called setEmployeeName, if you want to call the parent's setEmployeeName use super.setEmployeeName() if you want to call the child’s setEmployeeName (overridden setEmployeeName) use setEmployeeName() or this.setEmployeeName()

Covariant Return Type

  In Java 5.0 you can change the return type of an overridden method to any descendent class of that returned type. For example you can have a method called x and it returns an Employee object. You can have the same method x in a child class (overridden), but the return type can be an HourlyEmployee object. This is because a hourly employee is an employee.

public Employee getEmployeeName() // The base class method (original method)
public HourlyEmployee getEmployeeName() // The derived class method (still an overridden method)

  You can also change the overridden method’s access rights from restrictive to less restrictive, but not from less restrictive to more restrictive. More on this later.

Private ➡ Protected ➡ Public
(You can not go the other way)
Public x Protected x Private
